OPTIONAL: SETUP M-PESA PAYMENTS
═══════════════════════════════════════════════════════════════════

If you want to enable actual M-Pesa payments:

  1. Visit: https://developer.safaricom.co.ke/
  2. Create account → Create App
  3. Get these credentials:
     • Consumer Key
     • Consumer Secret
     • Passkey (Lipa na M-Pesa)
  4. Edit backend/.env:
     MPESA_CONSUMER_KEY=your_key
     MPESA_CONSUMER_SECRET=your_secret
     MPESA_PASSKEY=your_passkey
     MPESA_SHORTCODE=174379  # For sandbox
  5. Test M-Pesa:
     npm run test-mpesa

  See MPESA_SETUP.md for detailed instructions

TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════

❌ "Port 5000 already in use"
   → Change PORT in .env or kill the process using port 5000

❌ "Cannot connect to database"
   → Make sure PostgreSQL is running
   → Check DB_HOST, DB_PORT, DB_USER in .env
   → Try: psql -U postgres -d shule_sms

❌ "Invalid credentials at login"
   → Use: admin@vineyard.test / admin123456
   → If it doesn't work, run: npm run setup-db

❌ "M-Pesa test fails"
   → Check .env for M-Pesa credentials
   → Visit https://developer.safaricom.co.ke/ to get them

❌ "Database error"
   → Try: npm run setup-db
   → Or manually: psql -d shule_sms -f migrations/database.sql
